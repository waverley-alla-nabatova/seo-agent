# SEO + AEO Audit Agent

A multi-agent pipeline that crawls a website and produces a structured SEO + AEO audit report. Runs entirely inside **Claude Code** using your subscription — no Anthropic API key required.

## What it audits

| Area | What's checked |
|---|---|
| **Technical** | SSR gaps, canonical tags, redirect chains, broken links, large assets |
| **Metadata** | Titles, meta descriptions, H1s, OG tags — length, duplicates, missing |
| **Schema** | JSON-LD presence and validation per page type; generates missing blocks |
| **E-E-A-T** | Phone, email, address, testimonials, author bylines, LinkedIn signals |
| **AEO** | Question headings, answer chunk coverage, wall-of-text detection, citability |
| **PageSpeed** | Core Web Vitals scores, failing audits — mobile + desktop *(requires API key)* |

The report includes ready-to-use replacement titles, meta descriptions, and JSON-LD code blocks generated by Claude.

## Requirements

- [Claude Code](https://claude.ai/code) with an active subscription
- Python 3.14+
- [uv](https://docs.astral.sh/uv/) package manager
- [Playwright](https://playwright.dev/) for SSR rendering checks *(optional but recommended)*

## Installation

### 1. Clone the repo

```bash
git clone https://github.com/waverley-alla-nabatova/seo-agent
cd seo-agent
```

### 2. Install Python dependencies

```bash
uv sync
```

### 3. Install Playwright browsers *(for SSR gap detection)*

```bash
uv run playwright install chromium
```

### 4. Install the `/seo-audit` skill in Claude Code

Copy the skill file to your global Claude Code commands directory:

```bash
cp .claude/commands/seo-audit.md ~/.claude/commands/seo-audit.md
```

Then open the skill file and update the `scriptPath` on the last few lines to match where you cloned the repo:

```
scriptPath: /YOUR/PATH/TO/seo-agent/.claude/workflows/seo-audit.js
```

Also update the `ROOT` constant at the top of `.claude/workflows/seo-audit.js` to match your path:

```js
const ROOT = '/YOUR/PATH/TO/seo-agent'
```

The skill is now available in **any** Claude Code session — you don't need to open Claude Code from inside the `seo-agent` directory.

## Usage

Open Claude Code and run:

```
/seo-audit https://example.com
```

### Options

```
/seo-audit https://example.com --max-urls 50        # limit pages crawled (default: 200)
/seo-audit https://example.com --skip-render        # skip Playwright, faster but no SSR check
/seo-audit https://example.com --render-sample 5    # render N pages per type (default: 3)
/seo-audit https://example.com --skip-pagespeed     # skip PageSpeed section
/seo-audit https://example.com --output report.md   # custom output path
```

The report is written to `output/<domain>-audit.md` inside the `seo-agent` directory.

## Enabling PageSpeed data

The PageSpeed section requires a free Google API key.

### 1. Get a key

1. Go to [Google Cloud Console](https://console.cloud.google.com/)
2. Create a project (or use an existing one)
3. Navigate to **APIs & Services → Library**
4. Search for **PageSpeed Insights API** and enable it
5. Go to **APIs & Services → Credentials → Create Credentials → API key**
6. Copy the key

The PageSpeed Insights API is free with a quota of 25,000 requests/day.

### 2. Set the environment variable

Add it to your shell profile (`~/.zshrc`, `~/.bashrc`, or equivalent):

```bash
export PAGESPEED_API_KEY="your_key_here"
```

Then reload your shell:

```bash
source ~/.zshrc
```

That's it. The workflow reads `PAGESPEED_API_KEY` from the environment automatically — no changes to any files required. Run the audit normally and the PageSpeed section will be populated with real scores.

## Project structure

```
seo-agent/
├── audit.py                  # CLI entry point (discover / crawl / analyze)
├── modules/
│   ├── discovery.py          # Sitemap + robots.txt crawler, page-type classifier
│   ├── crawl.py              # HTML fetcher, selective Playwright rendering
│   ├── technical.py          # Canonicals, redirects, broken links, large assets
│   ├── metadata.py           # Titles, descriptions, headings, OG tags
│   ├── schema.py             # JSON-LD extraction and validation
│   ├── eeat.py               # Trust signal detection
│   ├── aeo.py                # Answer-engine readiness signals
│   ├── pagespeed.py          # PageSpeed Insights API
│   └── merge_rewrites.py     # Merges LLM-generated content into rewrites.json
├── models/
│   └── types.py              # Shared dataclasses
├── .claude/
│   ├── commands/seo-audit.md # /seo-audit skill definition
│   └── workflows/seo-audit.js # Orchestration workflow
├── .audit-cache/             # Per-run cache (gitignored)
├── output/                   # Generated reports (gitignored)
└── pyproject.toml
```
